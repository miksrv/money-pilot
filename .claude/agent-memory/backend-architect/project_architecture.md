## Auth Pattern (copy exactly)

```php
protected Auth $authLibrary;
public function __construct() {
    $this->authLibrary = new Auth();
}
// In each method:
if (!$this->authLibrary->isAuth) {
    return $this->failUnauthorized();
}
$userId = $this->authLibrary->user->id;
```

## ID Format
All PKs are `VARCHAR(15)` — short UUIDs generated by `generateId` callback in `ApplicationBaseModel`.

## Route Conventions
- All routes live inside `$routes->group('', ['namespace' => 'App\Controllers'], ...)` in `server/app/Config/Routes.php`.
- Every endpoint group requires matching OPTIONS routes for CORS preflight.
- `(:segment)` is used for path params.
- Nested resource routes use literal sub-paths: e.g., `(:segment)/archive`.

## Model Conventions
- Extend `ApplicationBaseModel`
- `$useAutoIncrement = false`, `$beforeInsert = ['generateId']`
- `$allowedFields` must explicitly include every writable column
- `$validationRules` uses `permit_empty` for nullable fields

## Response Conventions
- Use `ResponseTrait` methods: `respond()`, `respondCreated()`, `respondUpdated()`, `respondDeleted()`, `failUnauthorized()`, `failNotFound()`, `failValidationErrors()`, `fail()`
- `fail($data, $statusCode)` for custom 422s
- Consistent shape: `{ status, data, message }` from CI4 ResponseTrait

## Database
- MariaDB 10.5.8 at `127.0.0.1:3310` (Docker)
- Use `db_connect()` inline when raw query builder needed (no DI required)
- Count pattern: `db_connect()->table('x')->where(...)->countAllResults()`

## Known Model Relationships
- `transactions.payee_id` → `payees.id` (LEFT JOIN for payee name)
- `transactions.account_id` → `accounts.id`
- `transactions.category_id` → `categories.id`
- `transactions.user_id`, `accounts.user_id`, `categories.user_id` — multi-tenancy scope always required

## SessionModel
- Has `deleteByUserId(string $userId): void`

## UserModel
- `allowedFields = ['email', 'password', 'name', 'phone', 'is_active', 'last_login']`
- `beforeUpdate` hook hashes `password` automatically — pass plain text to `update()`
- User entity has `verifyPassword(string $password): bool`
- `created_at` may be a `\CodeIgniter\I18n\Time` object or string — handle both

## CategoryModel
- `getById(string $id): ?Category` — takes only id (no userId scope)
- `updateById(string $id, string $userId, array $data): bool`
- `deleteById(string $id, string $userId): bool`
- `findByUserIdWithSums()` returns categories with `expenses` property (current-month sum)

## AccountModel
- `getById(string $id, string $userId): array` — returns array (check `empty()`)
- `updateById(string $id, string $userId, array $data): bool`
- `deleteById(string $id, string $userId): bool`

## TransactionModel
- `getById(string $id, string $userId): array` — returns array (check `empty()`)

## Controller Gotcha — $model typed property
- `ResourceController` (the parent of `ApplicationBaseController`) declares `protected $model` without a PHP type.
- Adding `protected ModelClass $model` in a child controller causes a fatal PHP error: "Type of ... $model must not be defined".
- Solution: never type the `$model` property — just assign in constructor: `$this->model = new SomeModel()`.

## RecurringTransactionModel
- `getNextDueDate(string $frequency, string $from): string` — advances date by frequency using `\DateTime::modify()`
- `getDueToday(string $userId): array` — active rules with `next_due_date <= today`, end_date NULL or in future

## PayeeModel
- Global (no `user_id` FK) — `created_by_user_id` is metadata only, not a scope filter
- `getOrCreateByName(string $name, string $createdByUserId): string` — returns payee ID, increments `usage_count`
